<task type="auto">
  <name>Task 2: Create cloud-storage module and install @aws-sdk/client-s3</name>
  <files>server/cloud-storage.ts, package.json</files>
  <action>
1. Install the AWS SDK S3 client:
   ```bash
   npm install @aws-sdk/client-s3
   ```

2. Create `server/cloud-storage.ts` with the following implementation:

   - Import `S3Client`, `PutObjectCommand`, `DeleteObjectCommand` from `@aws-sdk/client-s3`
   - Import `crypto` from Node.js and `path` for extension extraction
   - Create a single `S3Client` instance configured with:
     - `region: 'auto'`
     - `endpoint: process.env.R2_ENDPOINT`
     - `credentials` from `R2_ACCESS_KEY_ID` and `R2_SECRET_ACCESS_KEY` env vars
   - Export three functions:

   `uploadFile(buffer: Buffer, originalFilename: string, mimeType: string): Promise<string>`
     - Generate a unique key: `${crypto.randomUUID()}${path.extname(originalFilename).toLowerCase()}`
     - Send PutObjectCommand with Bucket from `R2_BUCKET_NAME` env var
     - Return the public URL via `getPublicUrl(key)`

   `getPublicUrl(key: string): string`
     - Return `${process.env.R2_PUBLIC_URL}/${key}`
     - R2_PUBLIC_URL is the public bucket URL (e.g., `https://pub-xxx.r2.dev`)

   `deleteFile(key: string): Promise<void>`
     - Send DeleteObjectCommand for the given key

   Use non-null assertions (!) on env vars — the app will fail fast at startup if they are missing, which is the correct behavior. Do NOT add runtime checks that return empty strings.

3. Do NOT wire this module into routes.ts yet — that happens in Plan 02.
