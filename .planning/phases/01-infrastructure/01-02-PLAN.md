<task type="auto">
  <name>Task 2: Create migration script for existing local uploads</name>
  <files>script/migrate-uploads-to-r2.ts</files>
  <action>
Create `script/migrate-uploads-to-r2.ts` — a one-time Node.js script that migrates all existing local file references to R2 CDN URLs.

The script must:

1. Connect to the database using `DATABASE_URL` env var (create a standalone pg Pool — do not import from server/db.ts to avoid importing the full server).

2. Import `uploadFile` from `../server/cloud-storage` (reuse the same R2 module).

3. Query for rows needing migration:
   ```sql
   SELECT id, photo_url, storage_path FROM tagged_photos WHERE photo_url NOT LIKE 'https://%'
   SELECT id, photo_url, storage_path FROM photo_batch_items WHERE photo_url NOT LIKE 'https://%'
   ```

4. For each row, determine the local file path:
   - If `storage_path` or `photo_url` starts with `/uploads/` — file is at `${process.cwd()}/uploads/${filename}`
   - If `storage_path` starts with `/objects/` — log a warning (Replit GCS sidecar not available outside Replit; these files may not be recoverable). Skip and count as failed.

5. For each local file:
   - Read file with `fs.readFileSync(localPath)`
   - Determine MIME type from extension (`.jpg`/`.jpeg` = `image/jpeg`, `.png` = `image/png`, `.webp` = `image/webp`)
   - Upload to R2: `const newUrl = await uploadFile(buffer, filename, mimeType)`
   - Update the DB row: `UPDATE tagged_photos SET photo_url = $1 WHERE id = $2` (same for photo_batch_items)

6. Log progress:
   - `Migrating: {filename} -> {newUrl}`
   - Summary at end: `Migration complete: X succeeded, Y failed, Z skipped (GCS paths)`

7. Handle errors per-row (do not abort on a single failure). Log failures and continue.

8. Add a `--dry-run` flag that queries and reports what would be migrated without actually uploading.

Run with: `npx tsx script/migrate-uploads-to-r2.ts` (or `npx tsx script/migrate-uploads-to-r2.ts --dry-run`)

Required env vars: `DATABASE_URL`, `R2_ENDPOINT`, `R2_ACCESS_KEY_ID`, `R2_SECRET_ACCESS_KEY`, `R2_BUCKET_NAME`, `R2_PUBLIC_URL`
