<task type="auto">
  <name>Task 2: Create Stripe client, webhook handler with idempotency, and register route</name>
  <files>server/billing.ts, server/billing-webhook.ts, server/routes.ts</files>
  <action>
1. Create `server/billing.ts` — Stripe client instance and helper functions:
   ```typescript
   import Stripe from 'stripe';
   const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!, { apiVersion: '2024-12-18.acacia' });
   export { stripe };
   ```
   Also add a `getPlanTierFromPriceId` helper that maps a Stripe price ID back to 'starter' | 'pro' | 'free' by comparing against PLANS config. Return 'free' if no match.

2. Create `server/billing-webhook.ts` — full webhook handler following the research pattern exactly:
   - Import `stripe` from `./billing`, `db` from `./db`, `stripeEvents` and `users` from `@shared/schema`
   - `handleStripeWebhook(req, res)`:
     a. Read `stripe-signature` header
     b. Get rawBody via `(req as any).rawBody as Buffer` — do NOT use `req.body` or `JSON.stringify(req.body)`
     c. Call `stripe.webhooks.constructEvent(rawBody, sig, process.env.STRIPE_WEBHOOK_SECRET!)`
     d. On error, return 400 with error message
     e. Idempotency: SELECT from stripeEvents WHERE eventId = event.id. If exists, return 200 immediately
     f. INSERT into stripeEvents (eventId, type, processedAt) — do this BEFORE processing
     g. Switch on event.type:
        - `checkout.session.completed`: Extract session object, get `customer` and `subscription` IDs, get `metadata.userId`. Update users table with stripeCustomerId and stripeSubscriptionId. Then retrieve the full subscription via `stripe.subscriptions.retrieve()` and call syncSubscription.
        - `customer.subscription.updated`: Call syncSubscription with the subscription object
        - `customer.subscription.deleted`: Call syncSubscription (status will be 'canceled')
        - `invoice.payment_failed`: Extract customer ID, find user by stripeCustomerId, update subscriptionStatus to 'past_due'
     h. Always return 200

   - `syncSubscription(subscription)` helper:
     a. Get userId from subscription.metadata.userId. If missing, look up user by stripeCustomerId (subscription.customer as string)
     b. Call getPlanTierFromPriceId on subscription.items.data[0]?.price.id
     c. Update users: stripeSubscriptionId, subscriptionStatus (from subscription.status), planTier, trialEndsAt (from trial_end * 1000 or null), currentPeriodEndsAt (from current_period_end * 1000)

3. Register the webhook route in `server/routes.ts`:
   - Import `handleStripeWebhook` from `./billing-webhook`
   - Add `app.post('/api/stripe/webhook', handleStripeWebhook)` — place this near the top of `registerRoutes`, before any route-level middleware that might interfere
   - The rawBody is already captured globally in server/index.ts, so no special middleware needed on this route
