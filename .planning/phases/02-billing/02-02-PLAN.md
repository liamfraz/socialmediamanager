<task type="auto">
  <name>Task 1: Add checkout session, portal session, and billing status endpoints</name>
  <files>server/billing.ts, server/routes.ts</files>
  <action>
1. Add two functions to `server/billing.ts`:

   `createCheckoutSession(userId: string, priceId: string, email: string)`:
   - Look up user by ID from DB. If user has no stripeCustomerId, create a Stripe customer via `stripe.customers.create({ email, metadata: { userId } })` and save the ID to the users table.
   - Create Checkout Session via `stripe.checkout.sessions.create()` with:
     - `customer`: the stripeCustomerId
     - `mode: 'subscription'`
     - `payment_method_collection: 'always'` (card required for trial)
     - `line_items: [{ price: priceId, quantity: 1 }]`
     - `subscription_data: { trial_period_days: 14, metadata: { userId } }`
     - `success_url: process.env.APP_BASE_URL + '/billing/success?session_id={CHECKOUT_SESSION_ID}'`
     - `cancel_url: process.env.APP_BASE_URL + '/pricing'`
   - Return `session.url`

   `createPortalSession(userId: string)`:
   - Look up user by ID. If no stripeCustomerId, throw Error('No Stripe customer for this account')
   - Create portal session via `stripe.billingPortal.sessions.create({ customer: stripeCustomerId, return_url: process.env.APP_BASE_URL + '/dashboard' })`
   - Return `session.url`

2. Add three routes to `server/routes.ts` (inside registerRoutes, after the webhook route):

   `POST /api/billing/checkout` (requireAuth):
   - Validate body has `priceId` (string). Reject if missing.
   - Get userId from session, get user from DB for email
   - Call `createCheckoutSession(userId, priceId, user.username)` — username is used as email in this app
   - Return `{ url: sessionUrl }` (client redirects to this URL)

   `POST /api/billing/portal` (requireAuth):
   - Get userId from session
   - Call `createPortalSession(userId)`
   - Return `{ url: portalUrl }`

   `GET /api/billing/status` (requireAuth):
   - Get userId from session, look up user
   - Return `{ subscriptionStatus: user.subscriptionStatus, planTier: user.planTier, trialEndsAt: user.trialEndsAt }`
  </action>
  <verify>
    <automated>npx tsc --noEmit</automated>
  </verify>
  <done>POST /api/billing/checkout creates Stripe Checkout Session with 14-day trial and card required, POST /api/billing/portal creates Customer Portal session, GET /api/billing/status returns current subscription state</done>
</task>
