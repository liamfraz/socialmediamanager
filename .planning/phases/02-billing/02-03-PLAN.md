<task type="auto">
  <name>Task 1: Create requireSubscription middleware and apply to gated routes</name>
  <files>server/middleware/requireSubscription.ts, server/routes.ts</files>
  <action>
1. Create `server/middleware/` directory if it does not exist.

2. Create `server/middleware/requireSubscription.ts`:
   - Import `db` from `../db`, `users` from `@shared/schema`, `eq` from `drizzle-orm`
   - Export `requireSubscription` middleware function (req, res, next):
     a. Get userId from `req.session.userId`. If missing, return 401 `{ error: 'Not authenticated' }`
     b. Query user from DB by userId
     c. Check if `user.subscriptionStatus` is `'trialing'` or `'active'`
     d. If active: call `next()`
     e. If not active: return 402 with JSON:
        ```json
        {
          "error": "Subscription required",
          "upgradeUrl": "/pricing",
          "currentPlan": user.planTier || "free"
        }
        ```

3. Apply `requireSubscription` to feature-gated routes in `server/routes.ts`:
   - Import `requireSubscription` from `./middleware/requireSubscription`
   - Add `requireSubscription` as middleware to these existing routes (these are paid features):
     - POST `/api/posts/generate` (AI post generation — Starter+)
     - POST `/api/batches` (bulk upload with similarity detection — Starter+)
   - Do NOT gate basic CRUD routes (posts, photos, folders) — those work on Free tier
   - Add a comment `// Gated: requires active subscription` above each gated route for clarity
  </action>
  <verify>
    <automated>npx tsc --noEmit</automated>
  </verify>
  <done>requireSubscription middleware returns 402 with upgrade data for free/expired users, calls next() for trialing/active users, applied to AI generation and bulk upload routes</done>
</task>

   - Return `{ subscriptionStatus, planTier, trialEndsAt, isLoading, isActive }` where `isActive = status === 'trialing' || status === 'active'`

2. Create `client/src/components/UpgradePrompt.tsx`:
   - A reusable card/modal component shown when a user hits a 402 response
   - Props: `{ feature?: string }` (optional label for what they tried to access)
   - Shows: "Upgrade to unlock {feature || 'this feature'}"
   - Brief description of what paid plans include
   - Button: "View Plans" linking to `/pricing`
   - Use Tailwind + existing UI components (Card, Button from components/ui/)

3. Create `client/src/pages/Pricing.tsx`:
   - Import PLANS from shared/plans (the config, not the Stripe IDs — those are server-side)
   - Display three pricing cards side-by-side (responsive: 1 col on mobile, 3 cols on lg)
   - Each card shows: plan name, price/month (or "Free"), feature list with checkmarks, CTA button
   - Free tier: Button says "Current Plan" (disabled) if user is on free, or "Downgrade" otherwise
   - Starter tier: Button says "Start Free Trial" if user has never subscribed, "Current Plan" if on starter
   - Pro tier: Button says "Start Free Trial" if user has never subscribed, "Current Plan" if on pro
   - Subscribe buttons call `POST /api/billing/checkout` with the priceId and redirect to the returned URL
   - Use `useSubscription` hook to determine current plan and adjust button states
   - Highlight the recommended plan (Starter) with a subtle border/badge
   - IMPORTANT: PLANS config references process.env for priceIds which won't work client-side. Create a client-safe version of the plans config that includes everything EXCEPT priceIds. The priceId is sent from server config via the checkout endpoint — client just needs to know which tier to request. The checkout POST body should send `tier: 'starter' | 'pro'` and the server resolves the priceId from PLANS.

4. Add route in `client/src/App.tsx`:

   - Add `<Route path="/pricing" element={<Pricing />} />` — this should be accessible to both authenticated and unauthenticated users (it's a public page)

5. Update `server/routes.ts` checkout endpoint:
   - Change POST /api/billing/checkout to accept `{ tier: 'starter' | 'pro' }` instead of `{ priceId }` in the request body
   - Server resolves priceId from PLANS[tier].priceId
   - Reject if tier is 'free' or priceId is null (can't checkout free tier)
