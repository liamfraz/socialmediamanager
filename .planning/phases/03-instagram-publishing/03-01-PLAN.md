  **server/instagram.ts — Full rewrite.** Keep the same export interface names (InstagramAuthResult, InstagramPostResult) so routes.ts imports don't break. Keep postSingleImage, postCarousel, postToInstagram — but they will be updated in Plan 02 with container polling. For now, update their API base URL only.

  Changes:

  1. Replace constants at top:
     ```typescript
     const INSTAGRAM_APP_ID = process.env.INSTAGRAM_APP_ID!;
     const INSTAGRAM_APP_SECRET = process.env.INSTAGRAM_APP_SECRET!;
     const IG_AUTH_BASE = "https://api.instagram.com/oauth";
     const IG_GRAPH_BASE = "https://graph.instagram.com";
     ```
     Use non-null assertions (fail fast, per Phase 1 decision pattern). Remove the old `GRAPH_API_BASE`.

  2. Rewrite `getInstagramAuthUrl` to use Instagram Business Login:
     - URL: `https://api.instagram.com/oauth/authorize`
     - Scopes: `instagram_business_basic,instagram_business_content_publish`
     - Same params: client_id, redirect_uri, scope, response_type=code, state

  3. Rewrite `exchangeCodeForToken` to use Direct Login flow:
     - Step 1: POST to `https://api.instagram.com/oauth/access_token` with form-encoded body (client_id, client_secret, grant_type=authorization_code, redirect_uri, code). Returns { access_token, user_id }.
     - Step 2: Exchange short-lived for long-lived: GET `https://graph.instagram.com/access_token?grant_type=ig_exchange_token&client_secret=...&access_token=SHORT_TOKEN`. Returns { access_token, expires_in }.
     - Step 3: Get username: GET `https://graph.instagram.com/me?fields=username&access_token=LONG_TOKEN`. Returns { username }.
     - Return InstagramAuthResult with accessToken=long-lived token, instagramUserId=String(user_id), instagramUsername, expiresIn.
     - NO Facebook Page lookup. The user_id from Step 1 IS the Instagram user ID.

  4. Add `refreshInstagramToken` function:
     ```typescript
     export async function refreshInstagramToken(accessToken: string): Promise<{ accessToken: string; expiresIn: number } | null>
     ```
     - GET `https://graph.instagram.com/refresh_access_token?grant_type=ig_refresh_token&access_token=TOKEN`
     - Return new access_token and expires_in on success, null on failure.
     - This is called by the scheduler (Plan 03).

  5. Update postSingleImage, postCarousel to use `IG_GRAPH_BASE` instead of `GRAPH_API_BASE`:
     - Replace all `${GRAPH_API_BASE}` with `${IG_GRAPH_BASE}` in the publish functions.
     - Use `body: new URLSearchParams(...)` (form-encoded) instead of `JSON.stringify` with Content-Type application/json. The Instagram Graph API prefers form-encoded for mutations.
     - Container polling will be added in Plan 02 — do NOT add it here.

  **server/storage.ts — Add updateInstagramToken method:**

  1. Add to the IStorage interface:
     ```typescript
     updateInstagramToken(userId: string, accessToken: string, tokenExpiresAt: Date): Promise<void>;
     ```

  2. Implement in DatabaseStorage class:
     ```typescript
     async updateInstagramToken(userId: string, accessToken: string, tokenExpiresAt: Date): Promise<void> {
       await db.update(instagramCredentials)
         .set({ accessToken, tokenExpiresAt, updatedAt: new Date() })
         .where(eq(instagramCredentials.userId, userId));
     }
     ```

  3. Add `markInstagramTokenExpired` method for when refresh fails:
     ```typescript
     async markInstagramTokenExpired(userId: string): Promise<void> {
       await db.update(instagramCredentials)
         .set({ tokenExpiresAt: new Date(0), updatedAt: new Date() })
         .where(eq(instagramCredentials.userId, userId));
     }
     ```
     Add to IStorage interface as well.
