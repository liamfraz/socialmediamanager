This prompt requests a minimal, two-page UI focused entirely on the customer's process of reviewing, editing, and approving/rejecting social media posts generated by the AI system (n8n/Google Sheet backend).1. 🎨 General Design PrinciplesAesthetic Goal: Minimalist, clean, and highly professional, based on the provided "Flowtech" screenshot. Prioritize clarity and functionality over complex graphics.Colors: Use a white background and dark text. The only major color accent should be a bright blue (e.g., $#4285F4$) for primary actions and buttons (Approve, Login, etc.).Typography: Simple, readable sans-serif font (like Inter or Roboto).Structure: Responsive and easy to use on both desktop and tablet views. Use simple card components for visual separation of posts.2. 🔐 Login Page (Home)This is the only necessary page before the main dashboard.Purpose: Securely log in the customer using provided credentials.Layout: A single, central card-style container for the login form.Content:Logo/Title: Simple text title (e.g., "Post Review Dashboard").Input Fields: Username and Password.Button: A primary blue "Log In" button that redirects to the Dashboard upon successful (simulated) authentication.Requirement: The login should be designed to use hardcoded credentials provided by the developer initially.3. ✅ Post Review & Edit DashboardThis is the main, single-focus page where all action happens. The emphasis is on the review grid.A. Top Bar (Minimal)Logo/Title: Text title or logo in the top left.Action: A simple "Log Out" button in the top right.B. Optional: Post Generation Input (Simplified)Since the focus is on review, this section can be minimized or removed, but if included, it should be simple:A single, large text area for a "Post Idea/Prompt" (e.g., "Generate 5 posts about new spring flowers").A "Generate Posts" button (secondary style) to trigger the backend workflow.C. The Post Review Grid (Core Functionality)Display all pending posts in a clean, scrollable grid where each post is a dedicated, white, card component.Post Card Elements:Status Tag: A visual tag (colored pill) in the corner to show the current status:Review (Yellow)Ready to Post (Green)Posted (Gray)Caption/Text: A prominent, editable text area containing the AI-generated caption. This is the most important feature for the user to make quick edits.Image Preview Area:Show a few small, clear thumbnail images (e.g., 3-5 max).A concise label showing the total count (e.g., "1/18 Images").A small "View All" link/button that opens a modal window/lightbox to view all 20 images associated with the post.Action Buttons: Placed clearly at the bottom of the card."Approve" Button: The primary, bright blue button. Clicking this updates the backend status to "Ready to Post.""Reject/Archive" Button: A secondary (white background/blue border) button. Clicking this removes the post from the 'Review' queue or changes the backend status to 'Rejected'.4. 🔗 Data & Integration RequirementsThe UI must be designed to interface with the backend data (simulated or real).Dynamic Loading: The Post Review Grid must load and display data (Caption, Image URLs, and Status) from a simulated JSON array (representing the Google Sheet/database) on load.Action Handling: The "Approve" and "Reject" buttons must trigger a function that simulates sending an update back to the backend data source.
